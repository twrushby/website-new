I encountered some problems deploying the new site on Netlify. While the site was building successfully locally, it was failing during the Netlify deployment. The root of the problem appears to be that I am running a later version of Hugo than the one used by Netlify as default.

Luckily for me, a quick search on the error code generated by the deployment report returned a very useful post by Alison Hill.

Fixed by providing a build configuration file as per instructions provided by Alison Hill at https://alison.rbind.io/post/2019-02-19-hugo-netlify-toml/

Create a text file and then save the configuration file as netlify.toml in the root directory.

As soon as the config file was added, Netlify built the site successfully and it was published.
